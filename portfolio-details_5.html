About Projects

Title: DP World

Problem Statement:

In this use case study, I have to work on DP World Tool in which we have to create a tool that help to track and collect/maintain information of the containers off loaded from ship and loaded on trucks for further destination within India. The goal of DP world project ux design is to apply enhanced user experience and create a better overall DP world tool for the staff/Admin user experience to track the live status of the truck,sales, order created , Employee attendance and admin setting for configuring the tool features.

Requirements:

In order to design a DP world ux design, we need to understand the requirements of the staff/admin like what are the feature they planned or want as a tool to enhance their productivity in minimal efforts and with more security. This includes understanding what needs to be improved/implemented w.r.t to implement or integrate various APIs that need to be intgrated at the platform that have unique feature or operation to perform We have requirements like

  1. Need to create dashboard that shows real time data of the sales monthly wise as well as yearly wise according to the filter set by admin and total revenue
  2. Need to create order creation forms in order ot raise the order and that form have pre-requisites fields with mandatory fields
  3. Gather employee attendance data on the tool via punching machine data ,so need to show the same data in datatables and record the best performer and worst performer on the basis of attendance and order creation as well as sales done
  4. Need to show data of the container movement from offloaded to reaching the desired destination

User research:

User research is essential for understanding the needs of staff/Admins. This involves interviewing various staff and observing how they interact with the system ,under which age group they falls ,how much technical sounds they were, what are the problems they usually face, how they tackle the same during peak time. We want to know what are the things they need to track on regular basis and which information is crucial. The methods we used were:

  1. Competitive Analysis
  2. Surveys
  3. User Interviews
  4. Persona Development

For interview i interview future admins/reponsible person of the application and gather data like How many APIs we have, how many sensor are there, how they want data to be maintained. how they want data to be represented, filtered, what are the data that need to do analytics to show the same in more creative and unique way for better user experience.

Ideation & testing

I sketched ideas of how the Dashboard,live tracking ,gate information feature for Aviation project should look. Then created low fidelity wireframes and finally, high fidelity wireframes that incorporated the look and feel of Aviation project.I went through three rounds of testing; at each step of the design process i created a clickable prototype to test our ideas, obtained feedback from potential customers and iterated our design.

User Interface Design

Once i have a good understanding of the user needs, it's time to design the user interface. I need to create a user interface that is easy to use,provides a good user experience, have accessiblity,usablity,consitency in information flow, color theory should be aligned to company standard. I also need to make user interface for admins/staff who gonna track the data or use the data and maintained the live data as well as analysis the same. As a team contributor form Noida designing team for the web development i worked on deisgning the wireframe from low fidelity to high fidelity from doing requirement gathering to user research and analysis the data with a team of 3 designer and i lead the designer team for the prototyping. Following proces in agile and in iterative way i used to do designing in iteration wise and take the feedback from the stakeholder(airport staff) and the product owner of the application
