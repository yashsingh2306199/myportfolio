Title: Optimizing Truck Traffic Management on Terminal Level

Problem Statement:

A transportation company is facing challenges in managing the truck traffic between different terminals. The current system involves manual planning and coordination, which leads to inefficiencies and delays in the delivery of goods. The company wants to develop a new software system that automates the process of truck traffic management on the terminal level. Requirements:

Requirement 1: The truck traffic is executed on terminal level (e.g. Terminal A-Terminal B), please disregard pickup and distribution of load from the customer. The transportation company wants the new software system to focus only on the truck traffic management between different terminals. The pickup and distribution of the load from the customer will be handled separately and will not be part of the scope of this project.

Requirement 2: The user is entitled to select multiple terminals participating in the plan. The software system should allow the user to select multiple terminals that will participate in the plan. This will enable the user to optimize the truck traffic between different terminals, which will improve the efficiency of the transportation company.

Requirement 3: The user is entitled to provide information about every linehaulâ€™s distance and the time required to cover that distance. The user should be able to provide information about the distance between the different terminals and the time required to cover that distance. This information will be used by the software system to optimize the truck traffic between the terminals.

Requirement 4: It is preferred that all actions are performed in a single view without the necessity of using modal/pop-up windows. The software system should have a single view where the user can perform all the actions required for truck traffic management between different terminals. The use of modal/pop-up windows should be minimized to improve the user experience and simplify the process.

Solution:

To address the transportation company's challenges, our team developed a web-based software system that automates the process of truck traffic management on the terminal level. The software system includes the following features: Dashboard: The dashboard provides a single view of all the terminals participating in the plan. The user can select multiple terminals and view the current status of the truck traffic between them. Terminal Management: The user can add, edit, or delete terminals from the system. The distance between the terminals can also be specified, which will be used by the software system to optimize the truck traffic between them. Truck Traffic Management: The user can create a new truck traffic plan between the selected terminals. The software system will optimize the plan based on the distance and time required to cover that distance. The user can also view the current status of the truck traffic plan, which includes the estimated time of arrival and the actual time of arrival. Reporting: The software system generates reports on the truck traffic plan, which includes the estimated time of arrival, actual time of arrival, and any delays encountered during the transportation.

Conclusion:

The web-based software system developed by our team has helped the transportation company to automate the process of truck traffic management on the terminal level. The software system has improved the efficiency of the transportation company by optimizing the truck traffic between different terminals. The user can select multiple terminals, provide information about the distance and time required to cover that distance, and create a truck traffic plan. The use of modal/pop-up windows has been minimized, which has improved the user experience. The software system also generates reports on the truck traffic plan, which enables the transportation company to identify any delays encountered during the transportation.
