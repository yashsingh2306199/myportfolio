Project Overview


The COST tool was designed to replace Turbonomics, offering a customizable, cost-effective in-house solution to help enterprises track, analyze, and optimize Azure cloud resource consumption. Using Microsoft Azure APIs, it provides machine-level analytics, savings recommendations, and decision support.


Problem Statement

High dependency on third-party tools with poor flexibility and high costs (e.g., Turbonomics costing $10M/year) made cost tracking difficult. Our aim was to design a tool for internal deployment with API-driven intelligence and customizability.

Key Requirements

  • Search filters by VM, cost center, support group, etc.
  • Result views in both card and list format.
  • Drawer component with real-time cost analytics by request ID.
  • Policy creation pages for scaling operations.
  • User management access for admins.
  • Hierarchy/timeline view of request actions and exception handling.
  • Admin dashboard for request approvals and analytics.

User Research

User interviews revealed the need for a flexible, customizable interface that provided real-time feedback on savings opportunities. Key user needs included filtering large datasets, visualizing machine lifecycle, and tracking history of exception requests.

Methods used:

  • Competitive Analysis
  • Persona Development
  • User Interviews
  • Surveys

Information Architecture

  • Search & Explore: Filters, List/Card view, Cost Center Lookup
  • Drawer: Request ID details, New vs Old cost instances
  • Policy Engine: Create/edit scaling and savings policies
  • User Access: Admin user management panel
  • Timeline View: Actions, exceptions, request flow
  • Dashboard: Admin control panel for request oversight

Design & Testing

As lead designer, I guided the team through low- to high-fidelity prototypes using Adobe XD. Three test iterations were run to validate usability. Visual hierarchy and filtering were optimized for time-constrained admins.

UI Highlights

  • Dark theme with blue/purple accents for cloud tech feel
  • Quick toggles for card/list views
  • Component-based UI for customization
