IDMS – Intelligent Data Management System UX Case Study


Project Overview

IDMS is a platform designed to empower administrators with complete control over user access across physical touchpoints like gates, kiosks, restricted areas, and counters. The system focuses on creating a secure, centralized, and user-friendly dashboard experience for efficient monitoring and configuration.

Problem Statement

Manual tracking of user access across various airport points (gates, kiosks, boarding areas) was inefficient and risk-prone. Admins lacked a unified view of user access data, making it difficult to audit, configure, and act on real-time security changes. The goal was to design a digital console that streamlined this management with data-rich dashboards and filtering capabilities.

Requirements

  • Integrate APIs to manage and fetch live access data.
  • Display user access levels in a searchable, filterable data grid.
  • View and monitor real-time touchpoint activity (kiosks, gates, sensors).
  • Enable filtering by touchpoint, time, user role, or access zone.
  • Represent API activity visually for diagnostics.

User Research

We interviewed future users (admins and security personnel) to capture their routine interactions, issues faced during peak operations, and technical familiarity. Insights included:

  • Need for minimal-click navigation and real-time grid updates.
  • Desire for customizable alerts and visual diagnostics of API failures.
  • Importance of seeing company-user relationships at a glance.

Research methods:

  • Interviews with airport IT staff
  • Competitive Analysis of similar platforms
  • Surveys of potential admin users
  • Persona creation & journey mapping

Information Architecture

We organized the interface into high-utility categories:

  • Dashboard: High-level view of users and touchpoints
  • User Management: Grid view, filtering, access roles
  • Touchpoint Config: APIs mapping, terminal/sensor monitoring
  • Live Data: Streaming access attempts, logs
  • Diagnostics: API errors, response times, retry stats
  • 1. Dashboard
    • System status & overview
    • API activity alerts
  • 2. User Management
    • Filterable user grid with access roles
    • Search by company, zone, or name
  • 3. Touchpoint Configuration
    • Manage kiosks, sensors, gates
    • Map access levels to zones
  • 4. Live Data Tracking
    • Real-time access logs
    • Sensor activity visualized
  • 5. Diagnostics
    • API health status
    • Error logs and response latency
  • 6. Reporting
    • Export logs & performance charts
    • Filter by user, zone, date
  • 7. Settings
    • User roles & permissions
    • Alert configuration, timezone, etc.

Wireframes & UX Testing

Initial sketches were refined into low-fidelity and then mid-fidelity wireframes, which were shared in three rounds of feedback loops with IT staff. We tested:

  • Access grid filters and usability
  • Touchpoint visibility and zone grouping
  • Diagnostic alert readability

Final UI Design

The final UI focused on clarity, actionability, and performance. Key highlights:

  • Color-coded touchpoint statuses
  • Quick filter panel with saved filters
  • Live API performance charts and logs

Analytics & Reporting

Dashboards offer exportable reports showing usage trends, access anomalies, and API performance snapshots.

Conclusion

By understanding administrative pain points and designing with real-time needs in mind, we delivered an intuitive and scalable console that streamlines touchpoint security operations across a complex airport ecosystem.
